# 南鲸商城 — 需求文档

## 1. 项目概述

南鲸商城是一个专注于销售国货品牌的在线购物平台，为消费者、门店工作人员和商场管理人员提供商品浏览、购物车下单、库存管理和数据报表等全流程电商服务。

---

## 2. 角色划分

| 角色 | 说明 | 典型操作 |
|---|---|---|
| 游客 (Guest) | 未登录用户 | 浏览商店、查看商品、查看评论 |
| 顾客 (Customer) | 注册并登录的普通用户 | 下单、支付、退款、评价、领券 |
| 门店工作人员 (Staff) | 属于某一商店的员工 | 管理本店商品/库存/订单/优惠券、查看本店报表 |
| 商场管理员 (Admin) | 唯一的平台总管理员 | 创建商店、查看全局报表、发布全局优惠券 |

---

## 3. 功能模块列表

### 3.1 用户模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 用户注册 | 填写手机号、密码、昵称、身份注册账号 | 游客 |
| 用户登录 | 手机号 + 密码登录，按身份跳转不同页面 | 所有角色 |
| 修改密码 | 在个人信息页修改密码 | 顾客、Staff、Admin |
| 个人信息管理 | 修改昵称 | 顾客、Staff、Admin |

### 3.2 商品浏览模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 商店列表查看 | 展示平台内所有商店 | 游客、顾客 |
| 商店详情查看 | 进入商店后展示商品列表和商店信息 | 游客、顾客 |
| 商品详情查看 | 展示商品名称、价格、分类、库存、图片 | 游客、顾客 |
| 商品搜索 | 按关键词、品类、价格区间搜索商品 | 游客、顾客 |
| 分页展示 | 商品列表后端分页返回，不在前端全量加载 | 游客、顾客 |

### 3.3 商品管理模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 新建商品 | 填写名称、价格、分类（支持嵌套子类型）、库存、图片 | Staff |
| 编辑/删除商品 | 修改或下架商品 | Staff |
| 库存管理 | 更新商品库存数量（入库/出库） | Staff |
| 商品分类管理 | 支持父分类下建立子分类，如 食品 > 饮料 | Staff、Admin |

### 3.4 购物车模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 加入购物车 | 将商品加入购物车，可选数量 | 顾客 |
| 购物车管理 | 修改数量、删除商品 | 顾客 |
| 购物车结算 | 从购物车选择商品一次性下单 | 顾客 |

### 3.5 订单模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 创建订单 | 从购物车结算时生成订单，选择收货地址和优惠券 | 顾客 |
| 订单支付 | 对待支付订单进行支付（模拟支付） | 顾客 |
| 订单发货 | 对已支付订单标记为已发货 | Staff |
| 确认收货 | 对已发货订单确认收货，状态变为已完成 | 顾客 |
| 订单取消 | 未支付订单直接取消；已支付订单取消并退款 | 顾客 |
| 订单退款 | 待评价或已完成的订单申请退款 | 顾客 |
| 订单列表查看 | 顾客查看自己的订单；Staff 查看本店订单 | 顾客、Staff |

### 3.6 收货地址模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 地址管理 | 添加、编辑、删除多个收货地址和电话 | 顾客 |
| 下单时选择地址 | 结算时从已保存地址中选择 | 顾客 |

### 3.7 评论模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 查看评论 | 商品详情页展示所有评论及评分 | 游客、顾客 |
| 发表评论 | 对已购买并收货的商品发表评论和评分 | 顾客 |
| 回复评论 | 对任意评论发表回复（无需购买限制） | 顾客 |

### 3.8 优惠券模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 发布优惠券 | 创建优惠券组（折扣类型、有效期、数量） | Staff、Admin |
| 领取优惠券 | 在优惠券页面领取可用优惠券 | 顾客 |
| 使用优惠券 | 下单结算时选用已领优惠券 | 顾客 |

### 3.9 商店管理模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 创建商店 | 填写商店名称、统一社会信用代码、商店 Logo | Admin |
| 商店信息维护 | 修改商店基本信息 | Admin |

### 3.10 报表模块

| 功能点 | 说明 | 涉及角色 |
|---|---|---|
| 门店订单报表 | 查看/下载本店销售数据报表 | Staff |
| 全局汇总报表 | 查看所有商店订单的汇总报表 | Admin |

## 4. 数据库实体列表

| 实体 | 说明 |
|---|---|
| `user` | 用户，含角色字段（CUSTOMER / STAFF / ADMIN） |
| `store` | 商店，含名称、信用代码、Logo |
| `product` | 商品，含名称、价格、库存、图片、所属商店、分类 |
| `product_category` | 商品分类，自关联支持嵌套（parent_id 字段） |
| `cart` | 购物车，一用户对应一购物车 |
| `cart_item` | 购物车条目，含商品和数量 |
| `order` | 订单，含状态（待支付/已支付/已发货/已完成/已取消）、金额、地址 |
| `order_item` | 订单条目，含商品快照信息和数量 |
| `user_address` | 用户收货地址，一用户可有多条 |
| `review` | 评论，含评分、内容、商品关联，`parent_id` 支持楼层回复 |
| `coupon_group` | 优惠券组，含类型、面值/折扣、有效期、总数量、来源（店铺/全局） |
| `coupon` | 优惠券实例，属于某优惠券组，关联领取用户和使用状态 |

## 5. 暂不实现的功能

以下功能在当前阶段**不纳入开发范围**，避免过度开发：

| 功能 | 原因 |
|---|---|
| 推荐算法 | 依赖用户行为数据积累，算法设计复杂，属 Lab6 高难度加分项 |
| 客服功能（实时聊天） | 需要 WebSocket / 消息队列，架构复杂度高 |
| Redis 缓存 | 当前数据量不要求缓存层，可在性能瓶颈出现时再引入 |
| 未支付订单 14 天自动取消（定时器） | 需要 Spring Scheduler 独立模块，核心流程不依赖此功能 |
| 第三方支付对接（支付宝/微信） | 使用模拟支付替代，避免引入支付 SDK 复杂度 |
| 并发安全（优惠券超发防护） | 需要分布式锁设计，当前 MVP 阶段暂以数据库乐观锁兜底 |
| 订单报表文件导出（Excel/PDF） | 当前以接口返回 JSON 数据为主，文件导出后续扩展 |
| 多套优惠券叠加策略 | 基础实现支持单张优惠券使用，叠加逻辑后续扩展 |
