## 与文档 / 后端的已知差异

| 项 | 当前实现 | 应为（`api-spec.md`） |
|----|----------|------------------------|
| 登录字段 | `username` | `phone` |

## 建议实现顺序

```
1. 对齐 request.js + 登录/注册（phone、Result、存 role）
2. 三套 Layout + 路由（/、/staff、/admin）+ 角色跳转
3. 顾客浏览链路（商店列表 → 商店商品 → 商品详情 + 评论只读）
4. 购物车 + 地址
5. 订单全流程（创建 → 支付 → 发货/收货/取消/退款）
6. 优惠券（列表、领取、下单选用）
7. Staff 商品/订单/优惠券/报表
8. Admin 商店/全局优惠券/报表
```
