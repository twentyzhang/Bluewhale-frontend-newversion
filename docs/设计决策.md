## D-001 前端技术栈选型

| 项 | 内容 |
|----|------|
| **背景** | 需要主流、文档丰富、带组件库、支持 JWT 与热更新 |
| **决策** | React 18 + Vite + Ant Design 5 + Axios + React Router v6 |
| **理由** | 生态大、Ant Design 适合后台/商城类 UI；Vite HMR 体验好；与 Spring Boot REST 对接简单 |
| **备选** | Vue 3 + Element Plus（同样可行，团队更熟 React 则选 React） |
| **代价** | 包体积较大，后续可按路由做 code split |

## D-002 开发环境 API 走 Vite 代理

| 项 | 内容 |
|----|------|
| **背景** | 浏览器直连 `8080` 易触发 CORS |
| **决策** | 前端请求统一 `baseURL: '/api'`，由 `vite.config.js` 代理到 `http://localhost:8080` |
| **理由** | 开发体验与生产可配置反向代理一致；无需在后端为每个前端端口配 CORS |
| **生产** | 部署时由 Nginx / 网关将 `/api` 转发到后端，**不**在构建产物写死 `localhost:8080` |

## D-003 JWT 存 localStorage

| 项 | 内容 |
|----|------|
| **背景** | 需求与 `api-spec.md` 约定 Bearer Token |
| **决策** | Token 键名 `token`，存 `localStorage` |
| **理由** | 实现简单，刷新页面仍保持登录；课程/MVP 场景可接受 |
| **风险** | XSS 可窃取 token；生产应配合 CSP、输入消毒，长期可考虑 httpOnly Cookie + 后端刷新令牌 |
| **未选** | sessionStorage（关 tab 即登出，体验差）；内存 only（刷新丢失） |

## D-004 Axios 统一封装

| 项 | 内容 |
|----|------|
| **背景** | 所有接口需带 Token、统一错误与 401 处理 |
| **决策** | 单例 `src/api/request.js`，请求拦截器加 `Authorization`，响应拦截器处理业务码与 401 |
| **理由** | 页面不重复写 header；登录过期行为一致 |
| **约定** | 业务 API 按模块拆文件（`auth.js`、`order.js`…），均引用同一 `request` 实例 |

## D-005 路由守卫基于 token 存在性

| 项 | 内容 |
|----|------|
| **背景** | 需区分登录页与需登录页 |
| **决策** | `ProtectedRoute` / `PublicRoute` 仅检查 `localStorage` 是否有 `token` |
| **理由** | 实现快，与 JWT 无状态模型匹配 |
| **局限** | 不校验 token 是否过期、不校验角色；过期依赖接口 401 |
| **后续** | 增加 `RoleRoute`（读 `role`），Staff 路由校验 `storeId`（见 `api-spec` Staff 本店资源） |

## D-007 统一响应 Result 在前端解包（待实现）

| 项 | 内容 |
|----|------|
| **背景** | 后端 `Result<T>`：`{ code, message, data }` |
| **决策** | 在 Axios 响应拦截器解包：`code === 200` → 返回 `data`；否则提示 `message` 并 reject |
| **理由** | 页面层只处理业务数据，分页统一访问 `records` / `total` |
| **HTTP 错误** | 4xx/5xx 仍走 axios error 分支；body 内 `code` 非 200 走业务错误分支 |
| **状态** | 待实现 |

## D-008 状态管理暂不引入全局库

| 项 | 内容 |
|----|------|
| **背景** | 是否需要 Redux / Zustand |
| **决策** | MVP 阶段：`localStorage` + 登录信息 + React 局部 state；复杂后再引入 Context 或 Zustand |
| **理由** | 减少新手认知负担；购物车等可先用页面级 fetch + 刷新 |
| **触发重构** | 跨多级组件共享购物车数量、用户信息频繁变更时 |
