## 问题列表

### #001 登录请求字段与 API 不一致

| 项 | 内容 |
|----|------|
| **现象** | 登录接口返回 400 或校验失败，后端提示手机号相关错误 |
| **原因** | 前端 `Login.jsx` 使用表单字段 `username`，而 `api-spec.md` 要求 `phone` |
| **解决** | 将表单项 `name` 改为 `phone`，提交时 `{ phone, password }` |
| **状态** | ✅ 已解决（2026-06-03 用户模块） |
| **相关** | `src/pages/Login.jsx`、`docs/api-spec.md` 模块一 |

---

### #002 Axios 未解包 Result，导致取不到 token

| 项 | 内容 |
|----|------|
| **现象** | 登录接口 HTTP 200，但页面提示「未返回 token」 |
| **原因** | 后端统一响应为 `{ code, message, data }`；当前 `request.js` 拦截器返回 `response.data`（整包），页面从 `data.token` 取值的逻辑与真实结构不一致 |
| **解决** | 在响应拦截器中：若 `code === 200` 则 `return data`（业务体）；否则 `message.error` 并 `reject`。登录页使用 `const { token } = await request.post(...)` |
| **状态** | ✅ 已解决（2026-06-03 用户模块） |
| **相关** | `src/api/request.js`、`src/pages/Login.jsx` |

### #003 开发环境 API 404 或跨域

| 项 | 内容 |
|----|------|
| **现象** | 浏览器请求 `http://localhost:5173/api/...` 404；或直接请求 `8080` 出现 CORS 错误 |
| **原因** | 未启动后端；或未走 Vite 代理；前端写死了绝对地址 `http://localhost:8080` |
| **解决** | 1）先启动 Spring Boot（`8080`）；2）前端 `baseURL` 使用 `/api`；3）确认 `vite.config.js` 中 `proxy['/api']` 指向 `http://localhost:8080`；4）开发阶段不要绕过代理 |
| **状态** | ✅ 已配置代理 |
| **相关** | `vite.config.js` |

### #004 401 后无限跳转或 message 重复弹出

| 项 | 内容 |
|----|------|
| **现象** | Token 过期后多次弹出「登录已过期」或页面闪烁 |
| **原因** | 多个并发请求同时 401，每个都触发 `message.error` 和 `location.href` |
| **解决** | 可加「正在跳转登录」标志位，401 时只处理一次；或登录页请求不携带过期 token。当前实现已判断 `pathname !== '/login'` |
| **状态** | ⚠️ 简单防护已有，高并发场景可再优化 |
| **相关** | `src/api/request.js` |

### #006 ESLint react-refresh warning

| 项 | 内容 |
|----|------|
| **现象** | `npm run lint` 报 `react-refresh/only-export-components`：`router/index.jsx` 中内联了 `ProtectedRoute` |
| **原因** | 路由文件既导出 `router` 又定义组件 |
| **解决** | 将 `ProtectedRoute` / `PublicRoute` 抽到 `src/components/` 单独文件；或对该文件关闭该规则（不推荐首选） |
| **状态** | ✅ 已解决（路由组件已抽到 `src/components/`） |
| **相关** | `src/components/ProtectedRoute.jsx`、`PublicRoute.jsx` |
