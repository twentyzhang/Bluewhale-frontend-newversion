## 通用约定

### 分页列表

- 请求参数：`page`（默认 1）、`size`（默认 10）。
- 响应：`data.records`、`data.total`、`data.current`、`data.size`、`data.pages`。
- 前端 Ant Design `Table` / `Pagination`：`current={data.current}`，`pageSize={data.size}`，`total={data.total}`。

### 权限与角色

| 标记 | 前端处理建议 |
|------|----------------|
| 无需登录 | 不强制 token；可选登录态展示不同按钮 |
| 需要登录 | `ProtectedRoute` |
| 仅 Customer | `role === 'CUSTOMER'` |
| 仅 Staff | `role === 'STAFF'`，且 `storeId` 与路径 `{storeId}` 一致 |
| 仅 Admin | `role === 'ADMIN'` |

Staff 操作本店资源时，**优先用登录返回的 `storeId`**，不要仅信任前端传入的 storeId（路由可写死本店）。

### 金额与时间

- 金额：后端 `decimal`，前端展示用 `Number` 或格式化为两位小数，避免浮点累加误差（展示层 `toFixed(2)`）。
- 时间：ISO 字符串（如 `2026-05-20T10:00:00`），用 `dayjs` 格式化（可选依赖）。

## 模块一：用户

| 接口 | 细节 |
|------|------|
| `POST /api/auth/register` | `role` 仅 `CUSTOMER` / `STAFF`；Admin 不开放注册 |
| `POST /api/auth/login` | 请求 `{ phone, password }`；成功 `data`: `token`, `userId`, `nickname`, `role` |
| `GET /api/users/me` | Staff 的 `storeId` 有值；用于 Staff 端默认店铺 |
| `PUT /api/users/me/password` | 需 `confirmPassword` 与 `newPassword` 一致 |

**前端页面要点**

- 登录成功：写入 `token`，并建议缓存 `userId`、`nickname`、`role`（与 `设计决策.md` D-003 一致）。
- 跳转：见角色表（`CUSTOMER` → `/`，`STAFF` → `/staff`，`ADMIN` → `/admin`）。
- 手机号展示：接口可能脱敏 `138****8000`，编辑页勿用脱敏号回传。

**状态**：登录页 ⚠️ 未对齐；注册/个人中心 ⬜

## 模块二：商店与商品浏览

| 接口 | 细节 |
|------|------|
| `GET /api/stores` | 分页；`productCount` 用于列表展示 |
| `GET /api/stores/{storeId}/products` | 支持 `categoryId`、`keyword`、`minPrice`、`maxPrice` |
| `GET /api/products` | 全局搜索，响应格式同店内商品列表 |
| `GET /api/products/{productId}` | 含嵌套 `category`、`averageRating`、`reviewCount` |
| `GET /api/categories` | **树形数组**，非分页；适合 `TreeSelect` / 级联 |

**前端页面要点**

- 游客可访问列表与详情；「加入购物车」需登录且为 Customer。
- 商品详情库存 `stock`：加购时校验 `quantity <= stock`。
- 搜索：价格区间用两个 `InputNumber`，提交前校验 `minPrice <= maxPrice`。

**状态**：⬜ 未实现

## 模块三：商品管理（Staff）

| 接口 | 细节 |
|------|------|
| 新建/编辑商品 | `storeId` 来自当前 Staff 的 `storeId` |
| `PUT .../stock` | `type`: `IN` / `OUT`；`delta` 为正整数；返回 `currentStock` |
| `POST /api/categories` | `parentId` null 为顶级；Staff 与 Admin 可建 |
| `DELETE /api/categories/{id}` | **仅 Admin** |

**前端页面要点**

- 商品表单：分类用分类树选择叶子或任意节点（与后端约定一致）。
- 出库：前端校验不出库超过当前库存（最终以后端为准）。
- 图片：当前为 `imageUrl` 字符串，无上传接口时填 URL。

**状态**：⬜ 未实现

## 模块四：购物车

| 接口 | 细节 |
|------|------|
| `GET /api/cart` | 返回 `items` + `total`（非分页） |
| `POST /api/cart/items` | 同商品已存在则**数量累加** |
| `PUT /api/cart/items/{itemId}` | 改数量；注意库存上限 |
| `DELETE ...` | 删除单条 |

**前端页面要点**

- 仅 Customer；结算时勾选 `cartItemIds` 传给创建订单。
- `subtotal` 由后端算，前端展示即可。
- 失效商品（下架/库存不足）：创建订单时后端报错，需提示并刷新购物车。

**状态**：⬜ 未实现

## 模块五：订单

| 状态枚举 | 含义 | 顾客可操作 | Staff 可操作 |
|----------|------|------------|--------------|
| `PENDING_PAYMENT` | 待支付 | 支付、取消 | — |
| `PAID` | 已支付 | 取消（退款） | 发货 |
| `SHIPPED` | 已发货 | 确认收货 | — |
| `COMPLETED` | 已完成 | 退款 | — |
| `CANCELLED` | 已取消 | — | — |

| 接口 | 细节 |
|------|------|
| `POST /api/orders` | `cartItemIds`、`addressId` 必填；`couponId` 可选 |
| `POST .../pay` | 模拟支付，body `{}` |
| `POST .../cancel` | 未支付/已支付可取消；已支付 `refunded: true` |
| `POST .../confirm` | 仅 `SHIPPED` |
| `POST .../refund` | 仅 `COMPLETED`，需 `reason` |
| `POST .../ship` | Staff；需 `trackingNumber`、`carrier` |

**前端页面要点**

- 订单列表按 `status` Tab 筛选。
- 详情页按钮根据状态**动态显示**，避免非法操作（以后端校验为最终保障）。
- 创建订单响应含 `payableAmount`，可跳转支付或订单详情。

**状态**：⬜ 未实现

## 模块八：优惠券

| 类型 | 含义 | `value` 含义 |
|------|------|----------------|
| `DISCOUNT` | 折扣券 | 折扣率，如 `0.5` 表示 5 折 |
| `AMOUNT_OFF` | 满减券 | 减免金额 |

| 接口 | 细节 |
|------|------|
| `GET /api/coupon-groups` | `storeId` null 为全局券 |
| `POST .../claim` | 返回 `couponId` |
| `GET /api/coupons/mine` | `status`: `UNUSED` / `USED` / `EXPIRED` |

**前端页面要点**

- 领取：需登录 Customer。
- 下单：仅可选 `UNUSED` 且满足 `minOrderAmount` 的券。
- 展示：全局券与店铺券区分 `storeName`。
